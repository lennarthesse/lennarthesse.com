Creating a Personal Website (Part 2/2): The Web Server

August 10, 2022

In the first part of this series, we built our own website. Now we want to actually put it on the internet in a professional but still affordable way. This will allow you to have a professional online appearance beyond just having a website.

Serving to The Web

In order to serve a website to the world wide web, we need a so-called web server. It consists of a hardware and a software part, where the hardware can basically be any computer that is connected to the internet. For the software, we have several options like Apache or Nginx.

For this article I will be using Apache2. However, the setup for Nginx won't be much different. Additionally, I will use Ubuntu as the operating system.

Installing Apache

In Ubuntu, the installation is as simple as using the built-in package manager to download and install Apache. Note that we are using the root user and thus have all privileges. Otherwise, you will need to add sudo in front of the commands.

root@localhost:~# apt install apache2
Powered by Prism

Now everything should be set up and the web server should be running. To check, we can print out the contents of Apaches's configuration folder by typing ls /etc/apache2/, which should look like this:

root@localhost:~# ls /etc/apache2/
apache2.conf    magic           sites-available
conf-available  mods-available  sites-enabled
conf-enabled    mods-enabled
envvars         ports.conf
Powered by Prism

We can also request the status with systemctl. If the server is running, the output should look like the following. The relevant thing here is that it says Active: active (running).

root@localhost:~# systemctl status apache2.service
apache2.service - The Apache HTTP Server
    Loaded: loaded (/lib/systemd/system/apache2.service; enabled; vendor preset: enabled)
    Active: active (running) since Mon 2022-08-08 16:21:53 CEST; 3s ago
      Docs: https://httpd.apache.org/docs/2.4/
   Process: 12010 ExecStart=/usr/sbin/apachectl start (code=exited, status=0/SUCCESS)
  Main PID: 12014 (apache2)
     Tasks: 55 (limit: 18342)
    Memory: 5.1M
    CGroup: /system.slice/apache2.service
            ├─12014 /usr/sbin/apache2 -k start
            ├─12015 /usr/sbin/apache2 -k start
            └─12016 /usr/sbin/apache2 -k start
Powered by Prism

If it says inactive or failed, you can try to restart the server by using systemctl restart apache2.service.

Uploading The Website

If we now type the IP address of the computer running Apache (you can find it by typing hostname -I in the terminal) into our browser, we will see Apache's default page.

It gives some useful information about the structure of the configuration folder and, crucially for our next step, where the document root for our website's files is situated, being /var/www/. Here we can already create a folder for our website and copy all files into it.

Now we have to configure Apache to serve our website instead of the default page. Inside of the configuration folder /etc/apache2/ we change into the sites-available/ folder and copy the 000-default.conf file to a file with some name, e.g. our domain name, keeping the ending .conf.

Here we change/uncomment the following directives, replacing the [information] with our actual details.

<VirtualHost *:80>
    ServerAdmin [contact email address]
    ServerName [domain name]
    ServerAlias www.[domain name]
    DocumentRoot /var/www/[website folder]
</VirtualHost>
Powered by Prism

Both ServerName and ServerAlias are relevant if you have bought a domain, which I will talk about in a moment, and should be left out if there is no domain associated with the IP address.

We now enable our website's configuration, disable the default one, and restart the server with these three commands:

root@localhost:~# a2ensite [name].conf
root@localhost:~# a2dissite 000-default.conf
root@localhost:~# systemctl restart apache2.service
Powered by Prism

Now we have our website showing up.

Intermezzo

Up to this point, you could use your personal computer connected to your local network to serve your website. And it would be sufficient if you only wanted you and your family or roommates to see it.

To make it accessible for everyone connected to the internet, however, you would need to configure your router in a way to let outside traffic pass through. Furthermore, you would have to contact your internet provider and ask them for a static IP address since the address of your router that is actually connected to the internet can change.

While there are workarounds to obtain an effectively static IP address by constantly updating the address associated with the domain in the DNS record, I decided against this option. Simply renting a server will provide you with a static IP address, take care of making the server accessible from the outside, and often come with a professional email address and SSL certificate which allows traffic through the encrypted HTTPS protocol.

Renting a Server

At first, I was disappointed since all the hosting solutions I could find cost more than I was willing to spend. But after a little bit more research, I found that beside managed hosting, you can rent so-called root servers that simply provide a (virtual) machine running an operating system, leaving setup and configuration of the server to you, but costing significantly less.

So I went ahead and rented the smallest virtual private server at IONOS for 2€ a month and also bought a domain. To then set up Apache, we follow the steps I described above, connecting to the server through SSH.

To connect the domain to the IP address of our server, we add the domain and address to the DNS record through the server provider's control panel. Now we can actually add the ServerName and ServerAlias directives to our site's configuration and restart Apache for the changes to take effect. The server should now be accessible through our domain name.

Adding SSL

Now we want to allow traffic to use the encrypted HTTPS instead of HTTP. To enable HTTPS, we need an SSL certificate, which in my case was included in the root server package. First, we enable Apache's SSL module:

root@localhost:~# a2enmod ssl
Powered by Prism

Then we edit our website's configuration to support HTTPS traffic through port 443 and redirect any HTTP requests to HTTPS:

<VirtualHost *:80>
    ServerAdmin [contact email address]
    ServerName [domain name]
    ServerAlias www.[domain name]
    DocumentRoot /var/www/[website folder]
    ErrorLog ${APACHE_LOG_DIR}/error.log
    CustomLog ${APACHE_LOG_DIR}/access.log combined
    Redirect permanent / https://[domain name]
</VirtualHost>

<VirtualHost *:443>
    ServerAdmin [contact email address]
    ServerName [domain name]
    ServerAlias www.[domain name]
    DocumentRoot /var/www/[website folder]
    ErrorLog ${APACHE_LOG_DIR}/error.log
    CustomLog ${APACHE_LOG_DIR}/access.log combined
    SSLEngine on
    SSLCertificateFile      /path/to/certificate.cer
    SSLCertificateKeyFile   /path/to/private-key.key
</VirtualHost>
Powered by Prism

When setting up your SSL certificate through the provider, you will get access to a certificate and a private key file, which need to be linked with the SSLCertificateFile and SSLCertificateKeyFile directives.

Finally, we will configure a firewall to allow both HTTP and HTTPS traffic. Using UFW, you can see the status of the firewall by typing ufw status and available profiles by typing ufw app list. Upon installation, Apache creates preset profiles, so the output should look like this:

root@localhost:~# ufw app list
Available applications:
    Apache
    Apache Full
    Apache Secure
    OpenSSH
Powered by Prism

First, in order to ensure that we will still be able to connect to our server, we allow SSH traffic with ufw allow OpenSSH. Then, we allow the profile Apache Full which opens ports 80 for HTTP and 443 for HTTPS traffic. We then enable the firewall and check the status. The result should look like this:

root@localhost:~# ufw enable
root@localhost:~# ufw status
Status: active

To                  Action      From
--                  ------      ----
OpenSSH             ALLOW       Anywhere                  
Apache Full         ALLOW       Anywhere                  
OpenSSH (v6)        ALLOW       Anywhere (v6)             
Apache Full (v6)    ALLOW       Anywhere (v6)
Powered by Prism

And that's it already! We have set up our webserver and enabled SSL to allow a secure connection to our website.

Where to Go From Here

What I have shown here is still only a small part of what you can do. For example, it could be a good idea to use PHP to create more server-side interaction like loading data from a database, creating forms, or uploading files. Or you could dive deeper into JavaScript and add more fancy animations or host you own web app.

Still, a static website like the one we built is a good starting point for your online presence. Also, being able to have your own personalized email address helps you appear more professional and will definitely leave an impression on employers and other people researching you.

Creating a Personal Website (Part 1/2): HTML, CSS & JavaScript

August 3, 2022

In today's increasingly digitized world, having a proper digital presence is important. But what do you do if you don't want to feed your data to tech giants like Google or Facebook, or simply don't like today's social media networks?

This is exactly what I thought to myself just a few weeks ago. The answer is simple: build your own platform, independent and exactly fitted to your needs.

In this article series, I want to show the steps it took me to develop and afterwards host my own website, giving a rough outline of the entire process. You can take a look at the full source code on Github.

What We Will Need

The basic components of a website are HTML, CSS, and JavaScript. Additionally, if you want your website to be publicly available on the internet, you will need a web server to serve your page to visitors. For now, we will concentrate on building the website, leaving the setup of the webserver for part two.

HTML - The Basic Structure

HTML can be thought of as the backbone of a website. It structures the different parts and provides elements for CSS and JavaScript to attach to.

For my website, I wanted to have a large header filling the entire screen when you first visit the site. I also liked the idea of having everything on one site, not using any subsites, with a feed of all articles and a sidebar with additional information.

This left me with a basic outline of the HTML document's structure like this:

<!DOCTYPE html>
<html>
    <head>
    </head>
    <body>
        <header></header>
        <main>
            <article></article>
        </main>
        <aside>
            <div class="card"></div>
        </aside>
        <footer></footer>
    </body>
</html>
Powered by Prism

Next, it was time to prepare the individual elements for styling with CSS. For the header and the cards of the sidebar I basically just used regular headings and paragraphs. The articles, however, needed a more complex structure. I ended up adding an image, followed by a container with an article-head class for the title and date of the article, which will be styled differently.

<article>
    <img src="./path/to/image" alt="description">
    <div class="article-head">
        <h2>Title</h2>
        <small>Date</small>
    </div>
    <p>Article content</p>
</article>
Powered by Prism

CSS - The Looks

If you take a look at the website so far, you probably won't be convinced to let *that* be your online presence. This is where CSS comes in. It is used to style elements' attributes according to certain rules and will take our website from looking something like this:

...to looking more like this:

The first thing I did was to style the header. Since I wanted it to take up the entire screen, I set its height to 100vh, which means 100% of the vertical height of the viewport, the viewport being the part of the website that is currently visible.

For a complete reference of CSS units, HTML tags, and more I highly recommend W3Schools; it is where i got most of my information from and it provides interactive examples for basically everything.

After that, I set the background color, text color, and font size. To then center the text vertically, there are multiple solutions: the newer flexbox, and position/transform. I used the combination of position and transform, since flexbox only works on newer versions of browsers.

header {
    height: 100vh;
    background-color: #99ccdd;
}

header .container {
    color: white;
    position: absolute;
    top: 50%;
    transform: translate(0%, -50%);
}

header h1 {
    font-size: 4.5em;
}
Powered by Prism

I proceeded in a similar fashion with the other elements. To display the main content next to the sidebar, the float property can be used. This, however, can create complications with elements occluding each other. To fix this, the so-called "clearfix" hack can be used.

After a little bit more touching up, adjusting margins and paddings, and adding stylistic elements like borders, the bulk of the work is already done. Now we can move on to what brings our website from the ordinary to the exceptional.

JavaScript - Interaction, Animation, and More

Up to this point, our website is pretty static and doesn't do much. To add functionality when we press a button or create pop-ups and animations, we need JavaScript, the first real programming language in our trio.

In my case, I had to add a "Read More" button to the articles that would then expand the article, since I wanted to have everything on the main page. Firstly, I added a <button> element to the HTML document:

<div class="center-wrapper">
    <button class="centered colored-button">
        Read More
    </button>
</div>
Powered by Prism

I then styled it in CSS, centering it and adjusting a few attributes:

.center-wrapper {
    position: relative;
}

.centered {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
}

.colored-button {
    cursor: pointer;
    background-color: #4682b4;
    color: white;
    font-weight: bold;
    padding: 0.5em 1em;
}
Powered by Prism

Now, to add a function to the button, we can use its onclick event by writing <button onclick="our-javascript">. This will execute the JavaScript code inside the quotes. And while we could write it directly in there, it is better to call an earlier defined function that sits in a dedicated JavaScript file.

The function for my button looks like this:

function readMore(btn) {
    var parent = btn.parentElement.parentElement;
    var panel = parent.querySelector(".panel");

    if (panel.style.maxHeight == "0px") {
        panel.style.maxHeight = panel.scrollHeight + "px";
        btn.innerHTML = "Read Less";
    } else {
        panel.style.maxHeight = "0px";
        btn.innerHTML = "Read More";
    }                        
}
Powered by Prism

Upon call, the button passes itself to the function so that a variable of its parent, or rather grandparent, can be initialized. That element is actually the article that contains the button. This way, the function only targets the article whose "Read More" button has been pressed.

Then we check whether the article is collapsed, which is realized through a panel, containing the rest of the article, having a height of 0. If this is the case at the time the button is pressed, the panel is expanded to fit its content and the button's text is adjusted to "Read Less".

If the panel already is expanded, the function notices that the button is now a "Read Less" button and accordingly collapses the panel.

Animation

Animating an element is quite simple, too. At the bottom of the header, I wanted to have a hint that shows that there is more content waiting below. Again, I first added an HTML element, this time containing an icon, styled it, and then created a function in my JavaScript file.

The basic idea of an animation is to change an element's position by a small amount for every small timestep. Moving a positioned element can be done by changing one or more of the defining attributes top, bottom, left, and right.

Moving up or down can be realized by adding or subtracting from the bottom attribute. I added a boolean variable up, that changes at specified thresholds, and adjusted the position accordingly.

function animateHint() {
    if (xPos == 10) {
        up = false;
    } else if (xPos == 0) {
        up = true;
    }
    if (up) {
        xPos++;
    } else if (!up) {
        xPos--;
    }
    downIcon.style.bottom = xPos + "px";
}
Powered by Prism

This function is then called at a specified interval with setInterval(animateHint, 75);.

Wrapping Up

Having the basics laid down, it is only a matter of expanding on it. For example, I went on to add a different styling for small screens, where the sidebar is displayed underneath the articles, and a pop-up menu to navigate there. Also, I added more buttons and a search bar to look for specific articles.

The next step is to actually host our website on a server, which I will talk about in part two of this series.
